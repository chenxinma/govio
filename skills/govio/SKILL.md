# Govio 数据治理知识图谱

本 Skill 是数据治理查询的入口点，负责识别用户需求类型并路由到对应子 Skill。

## 子 Skill

| Skill | 用途 | 触发场景 |
|-------|------|----------|
| `govio-metadata` | 元数据查询 | 查询应用、表、字段、数据标准 |
| `govio-metrics` | 指标问数 | 查询指标数据、维度分析、趋势统计 |

## 需求路由

```
用户请求
  │
  ├─ 包含"指标"、"收入"、"签约"、"金额"、"YTD"等业务指标关键词
  │   └─ 路由到 govio-metrics
  │
  ├─ 包含"应用"、"表"、"字段"、"结构"等元数据关键词
  │   └─ 路由到 govio-metadata
  │
  └─ 不确定 → 先询问用户意图
```

### 路由规则

**路由到 `govio-metrics` 的场景**：
- 询问指标数据："本月账单收入是多少"、"签约覆盖率"
- 维度分析："按部门统计"、"按销售单元排名"
- 趋势分析："月度趋势"、"同比增长"
- 指标语义查询："这个指标怎么算的"、"指标来源"

**路由到 `govio-metadata` 的场景**：
- 应用查询："有哪些应用"、"财务管理领域有哪些系统"
- 表结构查询："AEP 有哪些表"、"T_INVOICE 表的字段"
- 数据资产盘点："列出所有表"、"哪些应用用了这张表"
- 数据标准查询："字段的数据标准是什么"

## 共享资源

两个子 Skill 共享以下资源：

```
assets/
├── schema.md            # 图模式（每次会话仅读一次）
├── metrics_index.md     # 指标索引
├── ontology.gml         # GML 元模型数据（NetworkX 后端）
└── names/               # 名称映射
     ├── node_names.md   #   全部节点名称汇总
     └── *.md            #   按应用分文件
```

## 名称解析

当 prompt 包含中文系统名（如"报价单中心系统""薪税系统"），**必须先 Grep 确认标准英文代码**：

- Grep 搜索 `assets/names/node_names.md`
- 或 Grep 搜索 `assets/names/` 下按应用分的文件

## 后端

后端类型由 `~/.govio/config.yaml` 决定（通过 `govio-cli onboard` 设置）。

| 后端 | 查询语言 | 使用场景 |
|------|---------|---------|
| `falkordb` | Cypher | govio-metadata / govio-metrics 的元数据查询 |
| `networkx` | Python | 本地 GML 文件查询 |

## Cypher 语法规范

- 属性值**必须用双引号**：`{name: "AEP"}` 而非 `{name: 'AEP'}`
- 必须以 `MATCH` 开头
- **必须包含 `LIMIT 300`**（明确需全量除外）
- Col 节点用 `column_name` 属性表示列名，不要用 `name`

## 查询终止策略

对同一语义目标最多 **3 次尝试**，逐步放宽：

1. 精确匹配（`name: "银行"`）
2. 模糊/包含匹配（`name CONTAINS "银行"`）
3. 同义词扩展（`金融`、`bank` 等）

3 次后仍无结果，**必须停止**并告知"知识图谱中未找到相关数据"。

## 排除场景

以下场景**不要**触发本技能：
- 数据迁移脚本编写
- 代码调试/修复
- 功能模块开发
- 数据库运维操作
- 数据比对、迁移验证 → 使用 `govio-observe`
