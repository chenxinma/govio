# 数据探源作业方法

> 本文档描述了系统性的数据探源和表关系梳理方法，适用于数据治理、系统迁移、数据血缘分析等场景。

---

## 📌 一、目的

1. **理解数据结构**：梳理业务系统涉及的所有数据表及其含义
2. **识别关联关系**：发现表与表之间的连接键和关系类型
3. **评估数据质量**：发现孤立数据、断裂链路、不一致问题
4. **输出可视化模型**：生成 ERD 图和网络关系图供沟通和文档使用

---

## 📋 二、适用范围

| 场景 | 说明 |
|------|------|
| 系统迁移 | 理解源系统数据结构，规划目标系统映射 |
| 数据治理 | 识别数据质量问题，制定清洗规则 |
| 数据血缘 | 追踪数据来源和流转路径 |
| 业务理解 | 通过数据结构反推业务流程 |
| 接口对接 | 确定数据交换的表和字段 |

### 步骤 2：读取表数据

**目标**：获取用于分析的数据样本或全量数据

```python
# 小表全量读取
df_quote = pd.read_sql("SELECT * FROM t_quote_info WHERE deleted = false", engine)

# 大表分页读取
def fetch_with_pagination(sql, engine, chunksize=50000):
    chunks = pd.read_sql(sql, engine, chunksize=chunksize)
    return pd.concat(chunks, ignore_index=True)

df_large = fetch_with_pagination("SELECT * FROM large_table", engine)

# 使用 Parquet 缓存（避免重复查询）
import polars as pl
df_cached = pl.read_parquet("./cache/large_table.parquet")
```

**注意事项**：
- 过滤软删除数据（`deleted = false`）
- 敏感数据脱敏处理
- 大数据量使用缓存

### 步骤 3：识别关联键

**目标**：分析表之间的连接关系

**常见关联模式**：

| 类型 | 示例 | 识别方法 |
|------|------|----------|
| 主外键 | `t_order.customer_id` → `t_customer.id` | 字段命名 convention |
| 业务编号 | `quote_number` = `osq_no` | 字段值对比 |
| 复合键 | `(emp_id, project_id)` | 多字段组合唯一性 |
| 跨系统映射 | `crm_quote_no` = `ocq_no` | 业务调研 + 数据验证 |

**验证方法**：

```python
# 检查外键关系是否存在
df_merge = df_child.merge(df_parent, left_on='parent_id', right_on='id', how='left')
orphan_count = df_merge['parent_id'].isna().sum()
print(f"孤立记录数：{orphan_count}")

# 检查复合键关联
df1_keys = df1.groupby(['emp_id', 'project_id']).size().reset_index()

common_keys = df1_keys.merge(df2_keys, on=['emp_id', 'project_id'])
```

### 步骤 4：构建关系图

**目标**：用图结构表示表间关系

```python
import networkx as nx

G = nx.Graph()

# 添加节点（表/实体）
for _, row in df_quote.iterrows():
    G.add_node(
        f"quote_{row['id']}",
        type="quote",
        quote_number=row.get('quote_number', ''),
        qid=row['id']
    )

# 添加边（关系）
for _, row in df_emp_prod_plan.iterrows():
    emp_node = f"emp_prod_{row['id']}"
    quote_node = f"quote_{row['quote_id']}"
    
    G.add_node(emp_node, type="emp_product", emp_id=row['emp_id'])
    
    if quote_node in G:
        G.add_edge(quote_node, emp_node, relationship="has_product")
```

**节点属性建议**：
- `type`：实体类型
- `id`：原始记录 ID
- 业务关键字段（用于后续分析）

```python
import matplotlib.pyplot as plt

plt.figure(figsize=(14, 10))

# 选择子图可视化（避免过大）
largest_cc = max(nx.connected_components(G), key=len)
subgraph = G.subgraph(list(largest_cc)[:200])

pos = nx.spring_layout(subgraph, k=0.5, iterations=50)

# 按类型着色
node_colors = []
for node in subgraph.nodes():
    node_type = subgraph.nodes[node].get('type', 'unknown')
    if node_type == 'quote':
        node_colors.append('lightblue')
    elif node_type == 'emp_product':
        node_colors.append('lightgreen')
    else:
        node_colors.append('red')

nx.draw_networkx_nodes(subgraph, pos, node_color=node_colors, node_size=50, alpha=0.8)
nx.draw_networkx_edges(subgraph, pos, alpha=0.5)
nx.draw_networkx_labels(subgraph, pos, font_size=6)

plt.title("实体关系网络图")
plt.axis('off')
```

### 步骤 6：连通分量分析

**目标**：发现数据完整性问题

```python
# 获取所有连通分量
subgraphs = [G.subgraph(c).copy() for c in nx.connected_components(G)]

# 统计每个子图的节点类型
subgraph_stats = []
for i, subgraph in enumerate(subgraphs):
    node_types = [attr.get('type', 'unknown') for node, attr in subgraph.nodes(data=True)]
    
    subgraph_stats.append({
        'subgraph_id': i,
        'total_nodes': len(subgraph.nodes()),
        'total_edges': len(subgraph.edges()),
        **{f"{t}_count": node_types.count(t) for t in set(node_types)}
    })

df_stats = pd.DataFrame(subgraph_stats).fillna(0)
```

**典型问题识别**：

```python
import polars as pl
```

## ⚠️ 六、注意事项

### 6.1 性能优化

```python
# 大数据量使用 Polars 而非 Pandas
import polars as pl
df = pl.read_parquet("large_file.parquet")

# 使用数据库端过滤
df = pd.read_sql("SELECT * FROM table WHERE condition", engine)

# 分页读取
for chunk in pd.read_sql("SELECT * FROM large_table", engine, chunksize=50000):
    process(chunk)
```

### 6.2 安全规范

- [ ] 使用密码管理服务，不硬编码密码
- [ ] 敏感数据脱敏后输出
- [ ] 配置文件权限设置为 600
- [ ] 不在日志中打印完整 SQL

### 6.3 常见问题

| 问题 | 解决方案 |
|------|----------|
| 内存不足 | 使用 Polars、分页读取、Parquet 缓存 |
| 图太大无法可视化 | 选取最大连通分量、限制节点数 |
| 关联键不唯一 | 检查数据质量、使用复合键 |
| 跨系统编号不一致 | 建立映射表、记录例外情况 |
