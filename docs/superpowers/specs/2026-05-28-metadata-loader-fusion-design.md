# Metadata Loader Fusion Design

## Overview

统一 `DatabaseLoader`（MySQL 元数据管理平台）和 `load_duckdb_metadata`（本地 DuckDB）两种元数据加载实现。抽象基类 `MetadataLoader`，`TDSLoader`（原 `DatabaseLoader`）和 `DuckDBLoader` 分别实现。`meta-export` 命令从 TDS 全量加载 + DuckDB 增量合并，输出与 `onboard` 一致的全量 CSV。

## Architecture

### Class Hierarchy

```
metadata/database.py
├── MetadataLoader (ABC)
│   ├── load_tables() -> pd.DataFrame     [abstract]
│   ├── load_columns() -> pd.DataFrame    [abstract]
│   ├── PhysicalTable -> pd.DataFrame     [property, delegates to load_tables]
│   └── Col -> pd.DataFrame               [property, delegates to load_columns]
├── TDSLoader(MetadataLoader)             [renamed from DatabaseLoader]
│   ├── __init__(db, workspace_uuid, schema_limits=None, app_names=None)
│   ├── app_names_map: dict | None        [stays on TDSLoader, not in base]
│   └── _convert_data_type(row) -> str    [internal, Oracle type conversion]

metadata/duckdb_loader.py
└── DuckDBLoader(MetadataLoader)
    ├── __init__(db_path, schemas)
    └── [implementation from current meta_export.py load_duckdb_metadata]
```

### Data Flow: `meta-export`

```
~/.govio/config.yaml
  kundb, workspace_uuid, app_list, app_map, relationship, metric
                    │
                    ▼
┌──────────────────────────────────┐
│         meta_export()            │
│                                  │
│  TDSLoader(kundb, ws, schemas)   │──► df_tables_tds, df_cols_tds
│  AppInfoLoader(app_list)         │──► df_apps
│  StandardLoader(kundb, ws)       │──► df_stds
│  DuckDBLoader(db_path, schemas)  │──► df_tables_duck, df_cols_duck
│                                  │
│  merge:                          │
│    df_tables = concat + dedup    │   (full_table_name, DuckDB 优先)
│    df_columns = concat + dedup   │   (column key, DuckDB 优先)
│    USE edges: merged tables      │   matched via app-map on schema
│                                  │
│  reorder_index([...])            │
│  write CSVs                      │
└──────────────────────────────────┘
```

### Output CSVs

Node CSVs (always generated):
- `PhysicalTable.csv` - merged TDS + DuckDB
- `Col.csv` - merged TDS + DuckDB
- `Application.csv` - from AppInfoLoader
- `Standard.csv` - from StandardLoader

Edge CSVs (always generated):
- `HAS_COLUMN.csv` - table→col from merged data
- `USE.csv` - app→table from merged tables + app-map

Edge CSVs (conditional, from config files):
- `RELATES_TO.csv` - if `relationship` in config
- `Metric.csv`, `Dimension.csv` - if `metric` in config
- `USES_TABLE.csv`, `REFERS_COLUMN.csv`, `DERIVED_FROM.csv`, `DIMENSION_USED.csv`, `SUPERSEDES.csv` - if metric loaded

Note: `COMPLIES_WITH.csv` is NOT generated by `meta-export`. It requires the recommender (`std-recommend` command) and depends on previously exported Col.csv + Standard.csv IDs.

### Merge Strategy

`merge_metadata` is a local helper in `meta_export.py`:

```python
def merge_metadata(df_tds: pd.DataFrame, df_duck: pd.DataFrame, key: str) -> pd.DataFrame:
    """TDS full + DuckDB incremental. DuckDB wins on conflict."""
    combined = pd.concat([df_tds, df_duck])
    return combined.drop_duplicates(subset=[key], keep="last")
```

- Tables: key = `full_table_name`
- Columns: key = `column` (dotted format `schema.table.column`)
- DuckDB rows come after TDS in concat, so `keep="last"` gives DuckDB precedence
- DuckDBLoader sets `data_entity_type = 'DUCKDB_TABLE'` / `'DUCKDB_COLUMN'` to distinguish from TDS types (`ORACLE_TABLE`, etc.)

### USE Edge for DuckDB Tables

DuckDB new tables need Application association. Reuse `app-map` from config:
- `app-map` maps `schema` → `name` (application name)
- Merged tables still have `schema` column
- Same `pd.merge` logic as current `make_csv` line 70-85

### 3. `src/govio/cli/meta_export.py` (rewrite)

```python
def meta_export(db_path: str, schemas: list[str], start_id: int, output: Path):
    # 1. Load config
    config = ConfigManager().load()

    # 2. Load TDS full metadata
    tds_loader = TDSLoader(config["kundb"], config["workspace_uuid"], schemas)
    app_loader = AppInfoLoader(config["app_list"], ...)
    std_loader = StandardLoader(config["kundb"], config["workspace_uuid"])

    # 3. Load DuckDB incremental
    duckdb_loader = DuckDBLoader(db_path, schemas)

    # 4. Merge tables and columns
    df_tables = merge_metadata(tds_loader.PhysicalTable, duckdb_loader.PhysicalTable, "full_table_name")
    df_columns = merge_metadata(tds_loader.Col, duckdb_loader.Col, "column")
    df_apps = app_loader.Application
    df_stds = std_loader.Standard

    # 5. reorder_index + write all CSVs (same logic as make_csv)
    # 6. Optionally load relationship/metric from config paths
```

## CLI Usage

```bash
# meta-export: DuckDB incremental + TDS full merge
govio-cli meta-export \
  --db ./local.duckdb \
  --schemas dm,dwd,dws \
  --start-id 1 \
  --output ./csv_output

# Reads from ~/.govio/config.yaml:
#   kundb, workspace_uuid, app_list, app_map, relationship, metric
```
